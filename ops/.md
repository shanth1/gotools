# `ops` Package

The `ops` package provides structured error handling primitives, allowing you to wrap errors with operation contexts (`Op`) and typed categories (`Kind`) for robust tracing and programmatic error checking.

## Usage

```go
package main

import (
	"errors"
	"fmt"
	"log"

	"github.com/shanth1/gotools/ops"
)

func main() {
	err := deleteUser(0)

	// Check error category using standard errors.Is
	if errors.Is(err, ops.KindInvalid) {
		log.Println("Bad request detected")
	}

	// Output: User.Delete: invalid: zero id
	fmt.Println(err)
}

func deleteUser(id int) error {
	const op = "User.Delete"

	if id == 0 {
		// Attach Operation and Kind to the error
		return ops.E(op, ops.KindInvalid, errors.New("zero id"))
	}

	return nil
}
```
